## Introduction

Effective project management depends on clarity. This includes clarity of timelines, budgets, progress, and risk.

This project transforms a raw project management dataset into an interactive, insight-driven Power BI dashboard designed to support strategic decision-making.

Rather than building a conventional report, I focused on creating a structured analytical experience. The dashboard walks users through:

- Portfolio-level performance

- Timeline tracking

- Detailed project-level analysis

To enhance usability and visual appeal, I designed the interface layout in PowerPoint before importing it into Power BI, ensuring a modern and intentional UI foundation.

The result is a clean, interactive, and business-ready dashboard that blends data storytelling with strong design principles.

## Dashboard Overview

This dashboard was designed as more than just a reporting tool. It was structured as a guided analytical journey.

Instead of overwhelming users with all metrics at once, the experience flows intentionally across three interconnected pages. Each page answers a different layer of business questions, gradually moving from strategic oversight to operational detail.

The interactive navigation buttons (custom-designed and icon-based) allow users to move seamlessly between sections, creating an intuitive, application-like experience rather than a static BI report.

### Page 1: Projects Summary: Understanding the Portfolio at a Glance
![Dashboard Page 1](/images/Image%201.JPG)

The Projects Summary page serves as the executive command center. It is designed for stakeholders who need clarity within seconds.

Upon landing here, users can instantly answer critical questions:

- How many projects are currently active?

- What percentage has been completed?

- Are we operating within budget?

- Where do risks appear to be concentrated?

Carefully structured KPI cards provide immediate visibility into overall performance, while supporting visuals reveal patterns in project status distribution and financial allocation.


### Page 2: Project Timeline: Where Time Tells the Truth
![Dashboard Page 2](/images/Image%202.JPG)

If the summary page shows what is happening, the timeline page reveals when and how efficiently it is happening.

Time is often the biggest risk factor in project management. Delays ripple across budgets, resources, and stakeholder expectations.

The Project Timeline page transforms raw start and end dates into visual insights:

- Projects mapped across time

- Duration comparisons

- Schedule gaps and overlaps

- Identification of potential bottlenecks

Instead of scanning spreadsheets of dates, users can visually detect patterns:

- Clusters of projects launching simultaneously

- Overextended timelines

- Delays relative to expected completion

This page helps shift conversations from reactive to proactive, making risks become visible before they escalate. 


### Page 3: Project Details – From Overview to Investigation
![Dashboard Page 3](/images/Image%203.JPG)

Once a trend or concern is identified, users can navigate to the Project Details page for deeper exploration.

This page was designed with interactivity at its core.

Through filtering and drill-through capabilities, users can:

- Select specific projects

- Compare budget vs actual performance

- Analyze resource allocation

- Examine performance metrics at a granular level

Here, the dashboard transitions from a storytelling tool into an analytical workspace.

Rather than presenting fixed insights, it empowers users to ask their own questions and discover answers dynamically.

It supports:

- Operational reviews

- Performance audits

- Strategic planning discussions

This final layer ensures the dashboard serves both executives and operational teams effectively.
